## Maintenance Convention Snapshot

Baseline conventions from docs index:

- `planning/board.md` is declared as active execution source of truth.
- `product/roadmap.md` is direction only and not task status detail.
- `product/specs/` is for focused specs.
- `references/` is for stable constraints.
- `history/` is not active todo tracking.
- `research/` is non-normative input.
- Active board wins when conflict exists with historical documentation.
